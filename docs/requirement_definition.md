# BookHub 要求定義書

## 1. プロジェクト概要・ビジョン
複数の電子書籍ストアに散らばった蔵書を一元管理し、ユーザーの「二度買い」を防ぐとともに「最も賢くお得な買い方」を提案するサービス。
ユーザーの財布と時間を守る**「漫画ヘビーユーザーのための専属執事（コンシェルジュ）」**として機能し、ユーザーファーストを徹底する。

## 2. ターゲットユーザー
* **メイン層**: 月に数千円〜数万円を電子書籍に費やす漫画ヘビーユーザー（クジラ層）。
* **主な特性**: 
    * ポイント還元やキャンペーンに合わせてAmazon、DMM、コミックシーモアなどを使い分けている。
    * 複数サイトを利用しているため、どこで何巻まで買ったか把握しきれていない。
    * PCで探し、スマホやタブレットで読むことが多い。
* **戦略的ターゲット**: 収益の柱として「成人向けコンテンツ」のユーザーを主力とし、リソースを集中。普及・宣伝の入り口として「一般向け」も展開する（本棚は成人向けと一般向けを分離）。

## 3. ビジネス要件と収益化モデル
### 3.1 収益化の基本方針
* ポップアップ等の「ユーザー体験を阻害する広告」は一切排除する。
* 「ユーザーの得になるコンシェルジュ的提案」を通じた高単価商材へのアフィリエイト誘導（CPA）をメイン収益とする。

### 3.2 マネタイズ機能
* **最安値シミュレーター（価格.com的比較）**: 「全巻買うなら、このストアの初回クーポンを使えば〇〇円お得」と計算・提案し、別ストアへ送客。※成人向けで先行・重点実装。
* **損益分岐点アプローチ**: 「動画サブスクの無料体験ポイントを使ったほうが実質タダ」等の提案によるサブスク誘導。
* **決済ルート最適化**: 購入のタイミングで、還元率の高いクレジットカードの作成を提案。

### 3.3 マイルストーン・KPI設定
* **アクティブユーザー想定**: 最大30万人規模を想定（Fanzaアクティブユーザー380万人の約5〜10%）。
* **利益目標（ARPU 2,750円/人 想定）**:
    * フェーズ1: 500人獲得 → 月10万円の利益
    * フェーズ2: 5,000人獲得 → 月100万円の利益
    * フェーズ3: 15,000人獲得 → 月300万円の利益
* **最終出口（Exit）**: 購買意欲が極めて高いトラフィック網を構築し、実績をもとにWebメディア企業やマーケティング会社へ事業譲渡（バイアウト）。

## 4. 機能要件
### 4.1 コア機能：自分だけの本棚（クロスデバイス）
* **蔵書の一元管理**: スマホ・PC問わず、全蔵書（成人向け・一般向けを完全に分離・隠蔽可能）を管理・閲覧。
* **スタック（本棚整理）機能**: 「〜編」など、ユーザー自身が自由に本棚を分けられる機能。
* **蔵書整理（ダメスタンプ・非表示機能）**: つまらなかった本や途中で読むのをやめた本に「×マーク」や「ブーイングスタンプ」を押し、ソート時に最下部へ移動、または非表示にする機能。

### 4.2 購買時の「防衛」と「ナビゲート」機能
* **二度買い防止警告**: 対象作品ページを開いた際、別ストアで所持していることを自動警告。
* **次巻の自動案内**: 所持している最新巻の「次の巻」を的確にナビゲート。
* **地雷回避（ステータス表示）**: 「連載中」「完結」に加え、「長期間休載中」「未完のまま数年経過」などのステータスを表示。
* **新刊・予約アラート**: 発売日（1週間前告知など）や特装版の予約開始日など、見逃せないタイミングでの通知。
* **積読の消化支援**: 未読作品を可視化し、週末などに読む提案を行う。

### 4.3 ランキング・比較機能
* 各ストアのランキングを横断して比較できる機能（週刊ランキングをベースに、日刊・月間も用意）。
* クーポンやキャンペーン情報と連動し「どこで買うのが一番お得か」を可視化（収益のコア機能だが、表向きは押し付けないUIとする）。

## 5. UI/UX・非機能要件
* **プラットフォーム**: まずはWebブラウザ版でリリースし、機能が安定・予算が確保でき次第、ネイティブアプリ化を検討する。
* **デザインの方向性**: 「本棚がきれいに整理できること」を軸とする。押し付けがましいデザインは避け、ユーザーファーストを徹底する。
* **視覚的なカタルシス**: 購入した本が本棚に一気に追加される際の「びっくり箱的」な視覚的演出（ON/OFF設定可能）。直感的に「どう使えばいいか」が目で見てわかるUI。

## 6. 将来の拡張要件（フェーズ2以降）
* **プロプランの導入**: 段階的に有料プラン（月額300円〜500円程度）を導入。
* **プレイリスト共有機能**: 自分の本棚（プレイリスト）を公開・共有できる機能（公開/非公開選択可）。
* **クリエイターファーストの還元システム**: 
    * 本ツール経由の総売上冊数を記録・可視化。
    * 合意した作家とインフルエンサー契約を結び、ツール経由のアフィリエイト収益の一部（例: 10%）を作家に還元。
* **ランキング紹介漫画/ショート動画**: ランキングを紹介する漫画を作成し表紙にスポンサーを表示。または、ショート動画から各ジャンルのランキングへ誘導。
* **ファンサイト的機能**: キャラクター辞典（ファンアート投稿）や、ユーザー評価（陪審員制度）によるサイト内ポイントの付与など。
